Проведи проверку вики проекта (`.claude/wiki/`) и её связи с задачами beads.
Воркфлоу и конвенции — `.claude/rules/wiki.md`.

## Что проверить

### 1. Целостность вики

- **Битые `[[…]]`.** Собери все ссылки: `grep -rho '\[\[[^]]*\]\]' .claude/wiki/`.
  Каждая `[[тип:slug]]` должна резолвиться в существующий файл
  `.claude/wiki/<тип-папка>/<slug>.md` (тип → папка: `module`→`modules`,
  `integration`→`integrations`, `concept`→`concepts`, `entity`→`entities`,
  `decision`→`decisions`). `[[overview]]` — на `overview.md`. `[[bead:…]]` —
  см. п. 3. Плоских `[[имя]]` без неймспейса (кроме `[[overview]]`) быть не должно.
- **Индекс.** Все ссылки `index.md` ведут на существующие файлы; новые страницы
  в индексе присутствуют.
- **Страницы-сироты.** Страницы без входящих `[[ссылок]]` (кроме `overview.md` и
  `decisions/`-страниц, на которые ссылаются через `wiki_refs`) — кандидаты на
  доработку перекрёстных ссылок.
- **Недостающие перекрёстные ссылки.** Страницы, упоминающие сущности друг
  друга, но не связанные `[[…]]`.
- **Frontmatter.** У каждой страницы есть `title`, `type`, `tags`, `sources`,
  `updated`; `type` соответствует папке.

### 2. Актуальность относительно кода

- Противоречия между страницами вики.
- Утверждения, устаревшие относительно текущего кода (выборочная сверка по
  `sources` из frontmatter). Истина — в коде.

### 3. Синхронизация с beads

- Для каждой задачи (`bd list --json`) поле `wiki_refs` в метаданных указывает
  на существующие файлы вики.
- Каждая ссылка `[[bead:map-xxxxxx]]` на страницах вики — на существующую
  задачу (`bd show map-xxxxxx`).
- Закрытые задачи (`bd list --status closed`) отражены страницей в `decisions/`
  с секцией `## Связанные Beads`.

## Итог

Выведи отчёт по каждой категории: что нашлось, где (файл:строка), как исправить.
Исправь однозначные проблемы (битые ссылки, индекс, frontmatter), о спорных —
сообщи. В конце допиши в `.claude/wiki/log.md` запись:

```
## [ГГГГ-ММ-ДД] lint | Краткий итог проверки
```
